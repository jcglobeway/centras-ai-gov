# Role-Based Core Flows

## 1. Operations Admin

목표는 기관별 서비스 상태를 한 번에 보고, 장애와 RAG 이슈를 빠르게 조치하는 것이다.

1. 운영 대시보드에서 기관별 상태와 KPI를 확인한다.
2. `error_rate`, `zero_result_rate`, `ingestion_failure`, `latency` 이상을 식별한다.
3. 기관 상세나 RAG 운영 화면으로 이동한다.
4. 문서 소스 상태, 최근 수집 이력, 검색 실패 질문을 확인한다.
5. 재수집, 재인덱싱, 이슈 등록, 고객사 알림 같은 운영 액션을 수행한다.
6. 조치 후 상태와 지표가 개선됐는지 확인한다.

## 2. Client Admin

목표는 자기 기관 챗봇이 민원 응대에 실제로 도움이 되는지 확인하고 개선 우선순위를 정하는 것이다.

1. 기관 대시보드에서 질문량, 해결율, 미응답률, 재문의율을 확인한다.
2. 질문 분석 화면에서 빈출 주제와 증가 추세를 확인한다.
3. 미해결 질문 목록에서 우선순위 높은 항목을 선택한다.
4. 질문 상세에서 질문, 답변, 출처, 검수 결과를 확인한다.
5. FAQ 추가, 문서 보완 요청, 담당 부서 전달, 재인덱싱 요청 중 적절한 액션을 수행한다.
6. 이후 동일 유형 질문의 품질 지표가 개선됐는지 확인한다.

## 3. QA / Knowledge Admin

목표는 오답과 미응답의 원인을 분류하고 개선 작업으로 연결하는 것이다.

1. 검수 대시보드에서 우선순위 높은 검수 대상을 확인한다.
2. 질문 원문, 답변, 검색 문서, 점수, 문서 상태를 함께 확인한다.
3. 원인을 `검색 실패`, `문서 없음`, `최신성 문제`, `생성 문제`, `정책 차단` 등으로 분류한다.
4. 검수 결과를 저장하고 FAQ 등록, 문서 수정 요청, 재인덱싱, 운영 이슈로 연결한다.
5. 반복 유형은 공통 개선 과제로 묶는다.
6. 조치 후 동일 원인군의 재발 여부를 추적한다.

## 4. Shared Event Checkpoints

- 질문 수신 이벤트
- 답변 결과와 출처 문서
- retrieval 로그와 zero-result 여부
- QA 검수 결과와 원인 코드
- 문서 보강, FAQ 등록, 재인덱싱 이력
- 기관 일별 KPI 집계 결과

세 역할은 화면은 분리되지만 같은 운영 로그 체계를 공유해야 한다.
