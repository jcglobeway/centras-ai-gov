# MVP PRD

## 1. Product Definition

공공기관용 멀티기관 RAG 챗봇을 `답변 서비스` 수준에서 끝내지 않고,
`운영`, `품질 개선`, `문서 최신화`, `성과 확인`까지 연결하는 운영 플랫폼으로 정의한다.

## 2. MVP Goal

- 시민 질문에 대해 근거 기반 답변을 제공한다.
- 운영사, 고객사, QA 담당자가 각자 필요한 운영 액션을 수행할 수 있게 한다.
- 질문 로그, 검색 로그, 검수 로그를 기반으로 품질 개선 루프를 운영할 수 있게 한다.

## 3. Primary Users

- `ops_admin`: 여러 기관 챗봇 상태, 문서, RAG 품질을 통합 운영
- `client_admin`: 자기 기관의 성과, 미해결 질문, 개선 포인트 확인
- `qa_admin`: 미응답, 오답 의심, 출처 문제를 검수하고 개선 작업으로 연결

## 4. MVP Scope

### In Scope

- 시민용 기본 RAG 챗봇
- 운영사 Admin 최소 기능
- 고객사 Admin 최소 기능
- QA 검수 화면 최소 기능
- 문서 등록, 수집, 재인덱싱 최소 기능
- KPI 집계 및 대시보드 최소 기능

### Out of Scope

- 고급 승인 워크플로우
- 질문 클러스터 자동화 고도화
- 복잡한 멀티테넌트 권한 상속 구조
- 자동 리포트 및 알림 고도화
- 고급 retrieval 정책 엔진

## 5. Core Problem

- 답변의 근거와 최신성 확인이 어렵다.
- 오답과 미응답을 체계적으로 개선하기 어렵다.
- 기관별 성과와 운영 품질을 분리해서 보기 어렵다.
- 문서 변경이 실제 답변 품질 개선으로 이어졌는지 추적하기 어렵다.

## 6. Core Operating Loop

1. 시민이 질문한다.
2. 챗봇이 검색과 생성으로 답변한다.
3. 질문, 답변, 검색 로그가 저장된다.
4. QA가 미응답과 오답 의심 건을 검수한다.
5. 운영사 또는 고객사가 문서 보강, FAQ 추가, 재인덱싱을 수행한다.
6. KPI 변화와 재발 여부를 확인한다.

## 7. MVP KPI

### Usage

- 총 세션 수
- 총 질문 수
- 순 사용자 수

### Resolution

- 명시적 해결율
- 추정 해결율
- 재문의율

### Quality

- 미응답률
- 오답 의심률
- 만족도
- 출처 포함 비율

### RAG Operations

- 검색 성공률
- zero-result rate
- 문서 없음 비율
- 수집 및 인덱싱 성공률

## 8. Success Criteria

- 기관 담당자가 미해결 질문과 개선 포인트를 주기적으로 확인할 수 있다.
- 운영사가 문서, RAG, 장애 상태를 한 화면에서 파악할 수 있다.
- QA가 검수 결과를 실제 개선 작업으로 연결할 수 있다.
- 챗봇 답변에 근거 출처를 표시할 수 있다.
- 핵심 KPI를 기관별로 집계할 수 있다.
