# Proposal: upload-page-v3

## Change ID

`upload-page-v3`

## Problem

현재 업로드 페이지(`/ops/upload`)는 기관/서비스를 선택하지 않아도 모든 입력 섹션이 활성화되어 있다. 컬렉션 이름은 자유 텍스트로 입력하는 구조여서 오타나 불일치가 발생하기 쉽고, 기존 컬렉션을 재사용할 수 없다. 리인덱싱 섹션은 개별 crawl_source를 나열하는 방식이어서 컬렉션 단위 운영이 어렵다.

## Proposed Solution

프론트엔드 `upload/page.tsx` 단일 파일만 변경한다. 백엔드 변경 없음.

1. **게이트 조건**: 기관(org)과 서비스(service)가 모두 선택된 이후에만 나머지 섹션이 활성화된다. 미선택 상태에서는 섹션을 흐리게(dimmed) 표시하고 입력을 막는다.

2. **컬렉션 피커 컴포넌트** (파일 업로드 섹션 및 웹 크롤 섹션 공통):
   - `GET /api/admin/crawl-sources?organization_id={orgId}` 호출 후 `serviceId` 기준으로 클라이언트 측 필터링
   - 결과에서 `collectionName` 유니크 값만 추출해 드롭다운으로 표시
   - 드롭다운 맨 아래에 "＋ 새 컬렉션 추가" 항목 제공
   - "새 컬렉션 추가" 선택 시 인라인 텍스트 입력 필드 표시

3. **파일 업로드 카드**: 컬렉션 피커를 파일 선택보다 앞에 배치한다.

4. **웹 크롤 카드**: 컬렉션 피커를 소스 이름/URL 필드보다 앞에 배치한다.

5. **리인덱싱 카드**:
   - 개별 소스 목록 대신 유니크 컬렉션 드롭다운으로 교체
   - 선택한 컬렉션에 포함된 소스 수를 "N개 소스 포함"으로 표시
   - 실행 버튼 클릭 시 해당 컬렉션의 모든 소스에 대해 `ingestionApi.runSource()` 를 순차 호출

## Out of Scope

- 백엔드 API 변경
- 컬렉션 삭제 버튼 (별도 change `collection-lifecycle`에서 처리)
- 컬렉션이 하나도 없을 때의 빈 상태 디자인 변경 (기존 유지)

## Impact

- 영향 파일: `frontend/src/app/ops/upload/page.tsx` 단일 파일
- 영향 API: 기존 `GET /api/admin/crawl-sources` 활용 (신규 API 없음)
- 영향 테스트: 프론트엔드 단위 테스트 없음 (현재 프로젝트 미구성)

## Done Definition

- 기관/서비스 미선택 시 파일 업로드, 웹 크롤, 리인덱싱 섹션이 모두 비활성(dimmed) 상태이다.
- 파일 업로드 섹션과 웹 크롤 섹션에서 기존 컬렉션을 드롭다운으로 선택하거나 새 컬렉션 이름을 인라인 입력할 수 있다.
- 리인덱싱 섹션에서 컬렉션 단위로 드롭다운을 선택하고 "N개 소스 포함" 카운트를 확인할 수 있다.
- 리인덱싱 실행 시 선택한 컬렉션의 모든 소스가 순차적으로 재인덱싱된다.
