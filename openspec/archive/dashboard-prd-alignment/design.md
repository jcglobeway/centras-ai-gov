# Design: dashboard-prd-alignment

## 설계 원칙

PRD Section 9의 핵심 원칙을 그대로 따른다.

1. **지표는 "조치 가능"해야 한다** — 숫자 노출만이 아니라 임계값 상태(ok/warn/critical)와 help 텍스트로 즉시 취해야 할 조치를 안내한다.
2. **운영사 / 고객사 화면은 분리한다** — 같은 데이터도 수신자에 따라 다르게 표현한다. 운영사는 시스템 건강도, 고객사는 민원응대 성과.
3. **지표는 "비교 가능"해야 한다** — 절대값 옆에 목표값(trendValue)을 함께 표시한다.

## 포털별 화면 설계

### Ops Admin (운영사) — `/ops`

```
[운영 대시보드]  기간 | 기관 필터

┌──────────┬──────────┬──────────┬──────────┬──────────┐
│ 응답률   │ Fallback │ 무응답률 │ 평균응답 │ 전체질문 │
│ 92.4%    │ 5.1%     │ 2.8%     │ 1,230ms  │ 1,245건  │
│ [정상]   │ [정상]   │ [정상]   │ [정상]   │          │
└──────────┴──────────┴──────────┴──────────┴──────────┘

┌──────────────────────────────────────────────────────┐
│ 기관 헬스맵                                          │
│  • org_seoul_120  응답률 88.2% (목표 90%)  [주의]    │
│  • org_busan_220  정상                      [정상]    │
└──────────────────────────────────────────────────────┘
```

**KPI 임계값 (운영사 기준)**

| KPI | 정상(ok) | 경고(warn) | 긴급(critical) |
|-----|----------|-----------|----------------|
| 응답률 | ≥ 90% | ≥ 80% | < 80% |
| Fallback율 | < 10% | < 15% | ≥ 15% |
| 무응답률 | < 5% | < 8% | ≥ 8% |
| 평균 응답시간 | < 1,500ms | < 2,500ms | ≥ 2,500ms |

**기관 헬스맵 상태 판정 로직**

```
resolvedRate < 80%  → critical ("응답률 X% (목표 90%)")
fallbackRate ≥ 15%  → critical ("Fallback율 X% (임계 15%)")
resolvedRate < 90%  → warn    ("응답률 X% (목표 90%)")
fallbackRate ≥ 10%  → warn    ("Fallback율 X% (임계 10%)")
그 외              → ok
```

### Client Admin (고객사) — `/client`

```
[기관 챗봇 성과 대시보드]  기간 | 기관 필터

┌──────────────┬──────────────┬──────────────┐
│ 총 문의 수   │ 자동응대율   │ 상담 전환율  │
│ 1,245건      │ 68.0%        │ 21.0%        │
│ (최신일)     │ [경고]       │ [경고]       │
└──────────────┴──────────────┴──────────────┘

┌──────────────┬──────────────┬──────────────┐
│ 평균 응답시간│ 재문의율     │업무시간외응대│
│ 1,230ms      │ 8.2%         │ 34.0%        │
│ [정상]       │ [정상]       │              │
└──────────────┴──────────────┴──────────────┘

[자동응대율 / 상담전환율 추세 차트]
```

**KPI 임계값 (고객사 비즈니스 기준, PRD Section 5)**

| KPI | 정상(ok) | 경고(warn) | 긴급(critical) | DB 컬럼 |
|-----|----------|-----------|----------------|---------|
| 자동응대율 | ≥ 70% | ≥ 60% | < 60% | `auto_resolution_rate` |
| 상담 전환율 | < 20% | < 30% | ≥ 30% | `escalation_rate` |
| 재문의율 | < 10% | < 15% | ≥ 15% | `revisit_rate` |
| 평균 응답시간 | < 1,500ms | < 2,500ms | ≥ 2,500ms | `avg_response_time_ms` |

> `auto_resolution_rate`, `escalation_rate` 등은 V023 마이그레이션으로 추가된 컬럼. 소수(0.68)로 저장되므로 프론트에서 ×100 변환.

### Client Failure 분석 — `/client/failure`

실패 원인 코드(A01~A10)를 PRD Section 6 기준으로 표시. 각 카드에:
- 코드 + 건수 Badge
- 원인 레이블 (한 줄)
- 상세 설명 (두 줄)
- 조치 주체 Badge (고객사/운영사/협의)
- 발생 비율 미니 바

| 코드 | 레이블 | 조치 주체 |
|------|--------|----------|
| A01 | 문서 없음 | 고객사 |
| A02 | 문서 최신 아님 | 고객사 |
| A03 | 파싱 실패 | 운영사 |
| A04 | 검색 실패 | 운영사 |
| A05 | 재랭킹 실패 | 운영사 |
| A06 | 생성 답변 왜곡 | 운영사 |
| A07 | 의도 분류 실패 | 운영사 |
| A08 | 정책상 제한 | 협의 |
| A09 | 질문 표현 모호 | 고객사 |
| A10 | 채널 UI 문제 | 운영사 |

| 저만족 응답 | 0건 | < 10건 | ≥ 10건 | `daily_metrics.low_satisfaction_count` |

## 컴포넌트 설계

### KpiCard 툴팁 위치

변경 전: `bottom-full left-1/2 -translate-x-1/2` → 화면 상단에서 뷰포트 위로 잘림
변경 후: `top-full left-0 mt-1.5` → 카드 아이콘 아래쪽으로 표시

```
┌──────────────────────────┐
│  LABEL ⓘ        [정상]  │
│                          │
│  92.4%                   │
│                          │
│  ▲ 목표 90%              │
│  ┌────────────────────┐  │
│  │ help 텍스트 설명   │  ← top-full: 카드 내부 아이콘 아래
│  └────────────────────┘  │
└──────────────────────────┘
```

### MetricsLineChart 지원 메트릭

| key | 레이블 |
|-----|--------|
| `resolvedRate` | 응답률(%) |
| `fallbackRate` | Fallback율(%) |
| `zeroResultRate` | 무응답율(%) |
| `avgResponseTimeMs` | 평균 응답(ms) |
| `autoResolutionRate` | 자동응대율(%) |
| `escalationRate` | 상담전환율(%) |
| `revisitRate` | 재문의율(%) |
| `afterHoursRate` | 업무시간외 응대율(%) |

## 데이터 흐름

```
PostgreSQL (daily_metrics_org)
  └─ V023 컬럼: auto_resolution_rate, escalation_rate, revisit_rate,
               after_hours_rate, knowledge_gap_count, unanswered_count,
               low_satisfaction_count

Spring Boot (MetricsController)
  └─ DailyMetricsSummary (domain) ← 이미 V023 필드 포함
  └─ DailyMetricsResponse (DTO)  ← 이번 변경으로 V023 필드 추가

Next.js (types.ts)
  └─ DailyMetric interface        ← 이번 변경으로 V023 필드 추가

페이지
  └─ /client → autoResolutionRate × 100 표시 (소수 → 백분율)
  └─ /ops    → metrics에서 per-org 집계 → 헬스맵
  └─ /qa     → lowSatisfactionCount 직접 표시
```
